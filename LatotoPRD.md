# 拉脱脱（Latoto）产品需求文档 PRD

---

# 1. 引言 / 概述（Introduction Overview）

## 1.1 产品名称

**拉脱脱（Latoto）**

## 1.2 产品类型

Android 极简相册整理工具。

## 1.3 产品背景

随着智能手机拍照能力不断提升，用户在旅行、聚会或日常生活中会产生大量照片。
这些照片中通常包含：

* 重复照片
* 模糊照片
* 截图
* 不满意的自拍
* 合照中表情不佳的照片

传统相册应用的删除流程较为繁琐，用户需要：

1. 选择照片
2. 进入多选模式
3. 删除确认

这一流程会导致用户**整理成本高、决策疲劳明显**。

## 1.4 产品定位

Latoto 是一款 **极简照片整理工具**，通过 **滑动手势交互**，帮助用户在短时间内快速筛选并删除不需要的照片。

产品设计原则：

* 极简
* 快速决策
* 无复杂管理
* 无学习成本

---

# 2. 产品目标（Goals / Objectives）

本产品目标符合 **SMART 原则**。

## 2.1 核心目标

帮助用户在 **5分钟内完成一次高效的照片筛选**。

## 2.2 产品目标指标

目标包括：

* 用户可在 **5分钟内浏览并决策 ≥200 张照片**
* 删除流程操作时间 **降低 70%**
* 用户平均每次使用删除 **30–100 张照片**

---

# 3. 目标用户 / 用户画像（Target Audience / Personas）

## 3.1 目标用户

普通智能手机用户。

用户特点：

* 拍照频率高
* 相册照片数量多
* 缺乏整理习惯
* 希望快速清理照片

---

## 3.2 典型用户画像

### 用户A：旅行记录者

特点：

* 旅行时一次拍摄大量照片
* 回家后需要筛选

需求：

* 快速删除模糊或重复照片
* 保留最满意的照片

---

### 用户B：日常拍照用户

特点：

* 经常拍自拍或合照
* 手机相册逐渐积累大量照片

需求：

* 快速筛选不满意照片
* 定期清理相册

---

# 4. 用户故事（User Stories）

### 用户故事 1：旅行照片整理

作为一个普通手机用户
当我旅行结束后拍了很多照片时
我希望可以快速浏览并删除不满意的照片
从而避免相册变得混乱。

---

### 用户故事 2：快速相册清理

作为一个手机用户
当我发现手机相册照片越来越多时
我希望用一种简单快速的方式进行筛选
从而在几分钟内清理掉不需要的照片。

---

### 用户故事 3：减少决策压力

作为一个整理照片的用户
当我不确定照片是否应该删除时
我希望可以先放入待定列表
从而稍后再做决定。

---

### 用户故事 4：避免误删

作为一个用户
当我误操作删除照片时
我希望可以快速撤销操作或从回收站恢复
从而避免丢失重要照片。

---

# 5. 用户流程（User Flow）

## 5.1 启动流程

```
打开 APP
   ↓
请求相册权限
   ↓
（首次使用）手势提示页
   ↓
进入整理模式
```

---

## 5.2 整理流程

```
浏览照片
   ↓
上滑 → 回收站
下滑 → 待定池
左滑 → 下一张
右滑 → 上一张
双击 → 撤销
```

用户可以 **随时停止浏览**。

---

## 5.3 待定池整理

```
进入待定池
   ↓
再次滑动决策
   ↓
上滑 → 删除
下滑 → 保留
```

---

## 5.4 回收站管理

用户可：

* 浏览删除照片
* 删除单张
* 多选删除
* 全部清空

---

# 6. 功能需求（Functional Requirements）

## 6.1 相册访问

功能：

* 读取系统相册照片
* 获取照片时间信息
* 获取照片缩略图

排序规则：

```
最新照片 → 最旧照片
```

---

## 6.2 整理模式

核心界面。

支持手势：

| 手势 | 功能    |
| -- | ----- |
| 左滑 | 下一张   |
| 右滑 | 上一张   |
| 上滑 | 移入回收站 |
| 下滑 | 加入待定池 |

---

## 6.3 撤销功能

操作：

```
双击当前照片
```

结果：

* 撤销上一操作
* 恢复照片状态

---

## 6.4 图片查看

支持：

```
双指缩放
```

用户可放大查看细节。

---

## 6.5 待定池

功能：

* 保存未决策照片
* 支持再次滑动整理

操作：

| 手势 | 行为 |
| -- | -- |
| 上滑 | 删除 |
| 下滑 | 保留 |

---

## 6.6 回收站

功能：

* 存储被删除照片
* 防止误删

支持：

* 删除单张
* 多选删除
* 全部清空

照片 **不会自动删除**。

---

## 6.7 页面导航

手势导航：

| 手势   | 页面  |
| ---- | --- |
| 双指上滑 | 回收站 |
| 双指下滑 | 待定池 |

返回方式：

```
Android 返回键
```

返回整理模式。

---

## 6.8 手势提示页

首次打开 APP 时显示。

内容包括：

* 左滑 / 右滑 浏览
* 上滑 删除
* 下滑 待定
* 双击 撤销
* 双指手势

用户点击后进入整理模式。

---

# 7. 非功能需求（Non-Functional Requirements）

## 7.1 性能

要求：

* 照片加载时间 < 200ms
* 滑动操作无卡顿
* 支持 ≥10000 张照片相册

---

## 7.2 设备支持

平台：

```
Android
```

最低版本建议：

```
Android 8.0+
```

---

## 7.3 权限

需要权限：

```
读取相册权限
```

权限请求：

* APP 启动自动请求

---

## 7.4 易用性

设计原则：

* 单手操作
* 无复杂菜单
* 无多层页面

---

# 8. 设计考量（Design Considerations）

界面原则：

* 极简 UI
* 照片全屏展示
* 手势为主交互

设计重点：

* 滑动动画流畅
* 操作反馈清晰
* 删除反馈明显
