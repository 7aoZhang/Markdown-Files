# Latoto 相册整理 APP

PRD V1.0

---

# 一、产品概述

## 1.1 产品定位

Latoto 是一款 **极简的相册整理工具**，帮助普通手机用户在 **5分钟内快速删除不需要的照片**。

核心方式：

**通过手势浏览 + 快速决策整理照片。**

---

## 1.2 核心用户痛点

普通用户的相册普遍存在问题：

* 照片数量过多
* 不知道哪些照片该删
* 手动选择删除成本高
* 相册管理过程枯燥

典型照片类型：

* 重复照片
* 模糊照片
* 截图
* 合照中不好看的照片
* 旅行后大量冗余照片

---

## 1.3 目标用户

普通手机用户

特征：

* 相册照片多
* 不使用复杂相册管理工具
* 希望快速清理空间

---

## 1.4 使用场景

用户通常在以下场景使用：

1️⃣ 一次旅行拍照后
2️⃣ 相册照片太多想整理
3️⃣ 手机空间不足

平均使用时长：

**约 5 分钟**

---

# 二、产品设计原则

## 2.1 极简原则

APP只保留核心功能：

* 浏览照片
* 快速决策
* 批量删除

不提供：

* 相册分类
* AI推荐
* 编辑功能
* 社交功能

---

## 2.2 无按钮原则

用户进入 APP 后：

**自动进入整理模式**

不需要：

* 点击开始
* 选择相册
* 手动加载

---

## 2.3 手势驱动

核心交互全部基于手势完成。

减少 UI 元素，提升操作效率。

---

# 三、核心功能


## 3.1 整理模式（核心功能）

用户打开 APP 后自动进入。

流程：

```
打开APP
↓
自动读取相册
↓
进入整理模式
```

用户通过浏览照片并进行决策。

---

## 3.2 照片浏览机制

照片以 **单张全屏卡片形式展示**。

浏览方式：

| 手势   | 行为    |
| ---- | ----- |
| 左滑   | 下一张照片 |
| 右滑   | 上一张照片 |
| 双指缩放 | 放大照片  |

---

### 浏览循环机制

当用户浏览到最后一张照片：

```
左滑
↓
回到第一张照片
```

实现 **无限循环浏览**。

---

## 3.3 照片决策操作

用户通过上下滑动进行照片处理。

| 手势 | 行为    |
| -- | ----- |
| 上滑 | 加入回收站 |
| 下滑 | 加入待定池 |

---

### 上滑删除交互

当用户上滑照片：

动画：

* 照片向上滑出屏幕
* 短暂滑出动画
* 手机轻微震动

---

### 下滑待定交互

当用户下滑照片：

动画：

* 照片向下滑出屏幕
* 短暂震动反馈

---

## 3.4 撤销操作

用户可以撤销最近一次操作。

触发方式：

```
双击当前照片
```

可撤销操作：

* 删除
* 待定

撤销结果：

恢复到上一状态。

---

# 四、照片管理结构

APP中照片存在三种状态：

```
相册照片
│
├── 整理中
│
├── 待定池
│
└── 回收站
```

---

# 五、待定池

待定池用于存放 **暂时不确定是否删除的照片**。

进入方式：

```
双指下滑
```

---

## 待定池浏览

| 手势 | 行为  |
| -- | --- |
| 左滑 | 下一张 |
| 右滑 | 上一张 |

---

## 待定池操作

| 手势 | 行为    |
| -- | ----- |
| 上滑 | 移入回收站 |
| 下滑 | 保留照片  |

---

# 六、回收站

回收站存放 **准备删除的照片**。

进入方式：

```
双指上滑
```

---

## 回收站操作

支持恢复和删除。

恢复：

| 操作   | 行为   |
| ---- | ---- |
| 恢复单张 | 返回相册 |
| 恢复多张 | 返回相册 |
| 恢复全部 | 返回相册 |

删除：

| 操作   | 行为   |
| ---- | ---- |
| 删除单张 | 系统删除 |
| 删除多张 | 系统删除 |
| 全部删除 | 系统删除 |

---

## 删除逻辑

重要规则：

```
只有在回收站删除的照片
才会真正从系统相册删除
```

用户在整理模式中的上滑操作：

只是 **加入回收站**。

---

# 七、照片整理会话机制

为了保证浏览顺序稳定，系统采用 **会话列表机制**。

流程：

```
进入整理模式
↓
系统读取当前相册
↓
生成整理会话列表
```

该列表在本次整理期间保持不变。

---

## 新照片处理

如果用户在整理过程中：

* 拍新照片
* 保存图片
* 产生截图

这些照片：

```
不会加入当前整理
```

只会在 **下次进入APP时出现**。
